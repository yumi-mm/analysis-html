# WebApp 充值关联 406 充值场景（source）

> 独立文档，不修改其它分析文档  
> 目的：在 **WebApp 端成功充值**（埋点 91）上，关联 **406 Web 充值** 埋点的 **`source`（充值场景）**  
> 时间范围示例：**2026-05-01～2026-05-31**（改 `params` 即可）

---

## 表与埋点说明

| 埋点 | log_id | 表 | 用途 |
|------|--------|-----|------|
| `RECHARGE_LOG` | **91** | `server_log_data_rt` | WebApp **充值成功**（金额；**WebApp 侧常无 `orderId`**） |
| Web 充值 | **406** | `server_log_data_rt` | WebApp **充值流程**（`source` 场景、`step` 步骤） |

**注意**：403～406 为 **业务埋点**（`type=1`），查 **`server_log_data_rt`**，**不要**查 `server_log_rt`。

### 406 extra 关键字段（按 step 区分）

| step | 含义（实测） | 可用字段 |
|------|------------|----------|
| **1** | 进入充值流程 | **`source`**、**`scene_type`**、`transaction_id` |
| **4** | **充值成功** | **`page`**（所在页面/场景）、`transaction_id`、`value`（USD）、`diamond`、`currency` |

> **同一单关联键**：`transaction_id`（如 `LP202606110955473541`）。  
> `step=4` 的 `source` 为空；`source` / `scene_type` **只在 step=1** 有值。

| 字段 | 说明 |
|------|------|
| `transaction_id` | **同一充值单主键**；step=4 JOIN step=1 用此字段 |
| `page` | step=4 用户所在页面路径（如 `/live/liveRoom`、`/message/detail`） |
| `source` | step=1 充值入口枚举（见下表 → `source_scene`） |
| `scene_type` | step=1 弹窗/子场景（见下表 → `scene_type_name`） |
| `value` / `diamond` | step=4 金额/钻石（本方案金额以 step=4 为准） |

## 关联逻辑（推荐：406 单内 step=4 ↔ step=1）

> **实测**：`step=4` 记录 `source` 全为空，但有 **`page`**；`source` / `scene_type` 仅在 **`step=1`**。  
> 因此：**不要**在 step=4 上直接读 `source`，也**不要**用 userid+时间窗乱配 406。

| 优先级 | 方案 | 关联键 | 说明 |
|--------|------|--------|------|
| **①（推荐）** | **page 场景** | `step=4` 的 `page` | 充值成功用户 = step=4；按 `page` 映射充值来源页面 |
| **②（推荐）** | **source / scene_type** | 先 `transaction_id`；失败则 **userid + 时间窗** | step=1 常**无** `transaction_id`（实测需兜底） |
| ③（兜底） | 91.extra | `source` / `rechargeSource` | 91 无 orderId；见方案 B |
| ④（已弃用） | 91 ↔ 406 时间窗 | userid + ±时间窗 | 旧版；step=4 无 source 导致匹配失败 |

### page → 场景名（① 用，可按 Step 9 增补）

| page（示例） | 场景名 |
|--------------|--------|
| `/live/liveRoom`、`/live/%` | 直播间 |
| `/videoChat/%`、`/videoChat` | 视频聊 |
| `/message/%` | 私聊 |
| `/group%` | 群聊 |
| `/party` | 交友房 |
| `/moment%`、`/feed%` | 朋友圈/随拍 |
| `/recharge` | 充值页（直达） |
| `/personal`、`/profile`、`/homePage`、`/home` | 个人页 |
| 未命中以上规则 | **其他**（不拆细 page） |

**限定 WebApp 用户（可选）**：406 无 `appType` 时，可与 91 WebApp 用户 INNER JOIN，或 `t_user_status type=10 value IN (36,40)`。

## step=4 充值总量（2026-05-01～2026-05-31）

> 口径：`log_id=406`、`step=4` 充值成功；金额 `value`（USD）；按 `transaction_id` 去重计单。

```sql
-- 2026年5月 step=4 充值用户 / 订单量 / 充值金额
WITH params AS (
    SELECT
        '2026-05-01' AS start_date,
        '2026-05-31' AS end_date,
        '20260501'   AS ds_start,
        '20260531'   AS ds_end
),
step4_raw AS (
    SELECT
        CAST(userid AS BIGINT) AS userid,
        GET_JSON_OBJECT(extra, '$.transaction_id') AS transaction_id,
        CAST(GET_JSON_OBJECT(extra, '$.value') AS DOUBLE) AS amount_usd,
        SUBSTRING(ts, 1, 19) AS recharge_time
    FROM chamet_data.server_log_data_rt
    WHERE ds >= (SELECT ds_start FROM params LIMIT 1)
      AND ds <= (SELECT ds_end FROM params LIMIT 1)
      AND log_id = '406'
      AND userid <> '1'
      AND TRIM(GET_JSON_OBJECT(extra, '$.step')) = '4'
      AND GET_JSON_OBJECT(extra, '$.transaction_id') IS NOT NULL
      AND TRIM(GET_JSON_OBJECT(extra, '$.transaction_id')) <> ''
      AND SUBSTRING(ts, 1, 10) >= (SELECT start_date FROM params LIMIT 1)
      AND SUBSTRING(ts, 1, 10) <= (SELECT end_date FROM params LIMIT 1)
      AND CAST(GET_JSON_OBJECT(extra, '$.value') AS DOUBLE) > 0
),
step4_dedup AS (
    SELECT userid, transaction_id, amount_usd, recharge_time
    FROM (
        SELECT *,
               ROW_NUMBER() OVER (
                   PARTITION BY transaction_id
                   ORDER BY recharge_time DESC
               ) AS rn
        FROM step4_raw
    ) t
    WHERE rn = 1
)
SELECT
    COUNT(DISTINCT userid) AS 充值用户数,
    COUNT(1) AS 订单量,
    ROUND(SUM(amount_usd), 2) AS 充值总金额_usd,
    ROUND(SUM(amount_usd) / COUNT(DISTINCT userid), 2) AS ARPPU_usd
FROM step4_dedup;
```

**按日拆分（可选）**：

```sql
-- 复用上方 CTE 至 step4_dedup，末尾替换为：
SELECT
    SUBSTRING(recharge_time, 1, 10) AS 充值日期,
    COUNT(DISTINCT userid) AS 充值用户数,
    COUNT(1) AS 订单量,
    ROUND(SUM(amount_usd), 2) AS 充值总金额_usd
FROM step4_dedup
GROUP BY SUBSTRING(recharge_time, 1, 10)
ORDER BY 充值日期;
```

### 与报表差距大？先看这里

**实测（2026-05）**：406 `step=4` 约 **51.2 万美金**（≈350 万人民币@7.0），而 `rpt_ly_webapp_d_report` 全月 `recharge_sum` 约 **800 万人民币** 量级——**不是 SQL 算错，是口径不同**。

| 差异来源 | 说明 |
|----------|------|
| **埋点 vs 报表** | 406 `step=4` = 客户端 **Web 充值漏斗**成功埋点；报表 `recharge_sum` = 数仓 **WebApp 日汇总**（偏财务/服务端，见 skill `rpt_ly_webapp_d_report`） |
| **币种** | 406 `value` 为 **USD**；报表 `recharge_sum` 多为 **人民币（元）**，`first_recharge_sum` 字段 COMMENT 已标「元」 |
| **覆盖范围** | 406 可能漏：未走完漏斗的成功单、部分渠道回调成功但客户端未打 step=4、重复 `transaction_id` 去重等 |
| **用途分工** | **金额对账** → 报表 / DWD / 91；**充值场景（page/source）** → 406 |

**建议**：场景分析继续用 406；汇报充值总额时以 **报表或 DWD** 为准，或按下面对账 SQL 说明差异比例。

## 主方案 ②：step=4 回溯 step=1 的 source / scene_type

> **实测**：仅 `transaction_id` 关联时 `has_step1_match` 常为 0——step=1 多数**没有**与 step=4 相同的 `transaction_id`。  
> v2：**优先 transaction_id**；未命中则用 **同一 userid、step=1 在 step=4 之前 ≤2 小时** 内最近一条有 `source` 的 step=1。

```sql
WITH params AS (
    SELECT
        '2026-06-01' AS start_date,
        '2026-06-11' AS end_date,
        '20260601'   AS ds_start,
        '20260611'   AS ds_end,
        7200         AS step1_lookback_sec   -- step=1 须在 step=4 前 2 小时内
),
step4_raw AS (
    SELECT
        CAST(userid AS BIGINT) AS userid,
        GET_JSON_OBJECT(extra, '$.transaction_id') AS transaction_id,
        GET_JSON_OBJECT(extra, '$.page') AS page,
        CAST(GET_JSON_OBJECT(extra, '$.value') AS DOUBLE) AS amount_usd,
        SUBSTRING(ts, 1, 19) AS recharge_time,
        UNIX_TIMESTAMP(SUBSTRING(ts, 1, 19)) AS recharge_ts
    FROM chamet_data.server_log_data_rt
    WHERE ds >= (SELECT ds_start FROM params LIMIT 1)
      AND ds <= (SELECT ds_end FROM params LIMIT 1)
      AND log_id = '406'
      AND userid <> '1'
      AND TRIM(GET_JSON_OBJECT(extra, '$.step')) = '4'
      AND GET_JSON_OBJECT(extra, '$.transaction_id') IS NOT NULL
      AND TRIM(GET_JSON_OBJECT(extra, '$.transaction_id')) <> ''
      AND SUBSTRING(ts, 1, 10) >= (SELECT start_date FROM params LIMIT 1)
      AND SUBSTRING(ts, 1, 10) <= (SELECT end_date FROM params LIMIT 1)
      AND CAST(GET_JSON_OBJECT(extra, '$.value') AS DOUBLE) > 0
),
step4_dedup AS (
    SELECT userid, transaction_id, page, amount_usd, recharge_time, recharge_ts
    FROM (
        SELECT *,
               ROW_NUMBER() OVER (PARTITION BY transaction_id ORDER BY recharge_time DESC) AS rn
        FROM step4_raw
    ) t
    WHERE rn = 1
),
-- step=1 全量（不要求 transaction_id，供时间窗兜底）
step1_all AS (
    SELECT
        CAST(userid AS BIGINT) AS userid,
        NULLIF(TRIM(GET_JSON_OBJECT(extra, '$.transaction_id')), '') AS transaction_id,
        NULLIF(TRIM(GET_JSON_OBJECT(extra, '$.source')), '') AS source,
        NULLIF(TRIM(GET_JSON_OBJECT(extra, '$.scene_type')), '') AS scene_type,
        SUBSTRING(ts, 1, 19) AS step1_time,
        UNIX_TIMESTAMP(SUBSTRING(ts, 1, 19)) AS step1_ts
    FROM chamet_data.server_log_data_rt
    WHERE ds >= (SELECT ds_start FROM params LIMIT 1)
      AND ds <= (SELECT ds_end FROM params LIMIT 1)
      AND log_id = '406'
      AND userid <> '1'
      AND TRIM(GET_JSON_OBJECT(extra, '$.step')) = '1'
      AND NULLIF(TRIM(GET_JSON_OBJECT(extra, '$.source')), '') IS NOT NULL
),
-- 有 transaction_id 的 step=1（精确关联）
step1_by_tid AS (
    SELECT userid, transaction_id, source, scene_type, step1_time
    FROM (
        SELECT *,
               ROW_NUMBER() OVER (PARTITION BY transaction_id ORDER BY step1_time ASC) AS rn
        FROM step1_all
        WHERE transaction_id IS NOT NULL
    ) t
    WHERE rn = 1
),
-- userid + 时间窗兜底：step=1 在 step=4 之前 N 秒内，取最近一条
step1_by_time AS (
    SELECT
        transaction_id AS s4_transaction_id,
        source,
        scene_type,
        step1_time
    FROM (
        SELECT
            s4.transaction_id,
            s1.source,
            s1.scene_type,
            s1.step1_time,
            ROW_NUMBER() OVER (
                PARTITION BY s4.transaction_id
                ORDER BY s1.step1_ts DESC

        FROM step4_dedup s4
        INNER JOIN step1_all s1
            ON  s4.userid = s1.userid
            AND s1.step1_ts <= s4.recharge_ts
            AND s4.recharge_ts - s1.step1_ts <= (SELECT step1_lookback_sec FROM params LIMIT 1)
    ) t
    WHERE rn = 1
),
joined AS (
    SELECT
        s4.userid,
        s4.transaction_id,
        s4.recharge_time,
        s4.page,
        s4.amount_usd,
        COALESCE(s1_tid.source, s1_time.source) AS source,
        COALESCE(s1_tid.scene_type, s1_time.scene_type) AS scene_type,
        COALESCE(s1_tid.step1_time, s1_time.step1_time) AS step1_time,
        CASE
            WHEN s1_tid.source IS NOT NULL THEN 'transaction_id'
            WHEN s1_time.source IS NOT NULL THEN 'userid_time'
            ELSE NULL
        END AS match_method,
        CASE COALESCE(s1_tid.source, s1_time.source)
            WHEN '2' THEN '速配余额不足'
            WHEN '3' THEN '约聊余额不足'
            WHEN '5' THEN '升级弹框'
            WHEN '6' THEN '礼物橱窗'
            WHEN '7' THEN '发现页'
            WHEN '8' THEN '游戏'
            WHEN '9' THEN '等级页'
            WHEN '10' THEN '个人页'
            WHEN '11' THEN '通知'
            WHEN '12' THEN '直播间浮层'
            WHEN '13' THEN '交友房浮层'
            WHEN '14' THEN '速配页'
            ELSE '其他'
        END AS source_scene,
        CASE COALESCE(s1_tid.scene_type, s1_time.scene_type)
            WHEN 'level_up' THEN 'v0限制弹窗'
            WHEN 'no_balance' THEN '余额不足弹窗'
            WHEN 'game' THEN '游戏'
            WHEN 'gift' THEN '送礼'
            WHEN 'live_collection_popup' THEN '直播间集合浮层'
            WHEN 'party_collection_popup' THEN '交友房集合浮层'
            WHEN 'video_chat_not_balance' THEN '视频聊余额不足'
            WHEN 'message' THEN '视频聊消息'
            WHEN 'video_chat_next_min_not_enough' THEN '视频聊下分钟钱不够'
            WHEN 'video_chat_free_end' THEN '试聊结束前余额不足'
            ELSE '其他'
        END AS scene_type_name
    FROM step4_dedup s4
    LEFT JOIN step1_by_tid s1_tid
        ON s4.transaction_id = s1_tid.transaction_id
    LEFT JOIN step1_by_time s1_time
        ON s4.transaction_id = s1_time.s4_transaction_id
        AND s1_tid.transaction_id IS NULL

SELECT
    userid,
    transaction_id,
    recharge_time,
    page,
    amount_usd,
    source,
    source_scene,
    scene_type,
    scene_type_name,
    step1_time,
    match_method,
    CASE WHEN source IS NOT NULL THEN 1 ELSE 0 END AS has_step1_match
FROM joined
ORDER BY recharge_time DESC;
```
