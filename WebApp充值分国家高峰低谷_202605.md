# WebApp 充值分国家高峰/低谷分析（2026-05）

> 独立文档，不修改 `WebApp充值24小时趋势_202605.md`  
> 口径：埋点 **`server_log_data_rt` + `log_id=91`**，WebApp（`appType/apptype` 36/40）  
> 时间：**2026-05-01～2026-05-31** 全月充值订单  
> 时段：**高峰期 0～5 点**、**低谷期 8～10 点**（库内时区，按充值发生时刻的小时）  
> **国家：不限定国家列表，统计该时段内所有有充值用户的国家**

---

## 指标说明

| 指标 | 计算方式 |
|------|----------|
| **ARPPU** | 该国在该时段内充值总金额（USD）÷ 该国在该时段内去重付费用户数 |
| **用户占比** | 该国在该时段内去重付费用户数 ÷ **该时段全部国家合计**去重付费用户数 × 100% |

### 用户国家字段

| 表 | 字段 | 说明 |
|----|------|------|
| **`chamet_data.dim_user_f_d`** | **`country_name`** | 用户维度国家名（与数仓画像一致） |

- 关联：`dim_user_f_d.userid = 充值用户 userid`，且 **`dim_user_f_d.ds = user_ds_snapshot`**
- 默认 `user_ds_snapshot = '20260531'`（5 月末快照；执行前确认分区已产出）

```sql
COALESCE(NULLIF(TRIM(uf.country_name), ''), '未知') AS country_name
```

## 主查询：全国家宽表（高峰 / 低谷 ARPPU + 用户占比）

```sql
-- WebApp 埋点91 · 全国家 · 高峰(0-5) / 低谷(8-10)
WITH params AS (
    SELECT
        '2026-05-01' AS start_date,
        '2026-05-31' AS end_date,
        '20260501'   AS ds_start,
        '20260531'   AS ds_end,
        '20260531'   AS user_ds_snapshot,  -- dim_user_f_d 快照批次
        '91'         AS log_id
),
log_raw AS (
    SELECT
        CAST(userid AS BIGINT) AS userid,
        GET_JSON_OBJECT(extra, '$.orderId') AS order_id,
        COALESCE(
            GET_JSON_OBJECT(extra, '$.createTime'),
            GET_JSON_OBJECT(extra, '$.payTime'),
            ts
        ) AS recharge_time_str,
        CAST(COALESCE(
            GET_JSON_OBJECT(extra, '$.money'),
            GET_JSON_OBJECT(extra, '$.rechargeMoney'),
            GET_JSON_OBJECT(extra, '$.price')
        ) AS DOUBLE) AS amount_usd
    FROM chamet_data.server_log_data_rt
    WHERE ds >= (SELECT ds_start FROM params LIMIT 1)
      AND ds <= (SELECT ds_end FROM params LIMIT 1)
      AND log_id = (SELECT log_id FROM params LIMIT 1)
      AND userid <> '1'
      AND (
          GET_JSON_OBJECT(extra, '$.appType') IN ('36', '40')
          OR GET_JSON_OBJECT(extra, '$.apptype') IN ('36', '40')
      )
      AND SUBSTRING(COALESCE(
              GET_JSON_OBJECT(extra, '$.createTime'),
              GET_JSON_OBJECT(extra, '$.payTime'),
              ts
          ), 1, 10) >= (SELECT start_date FROM params LIMIT 1)
      AND SUBSTRING(COALESCE(
              GET_JSON_OBJECT(extra, '$.createTime'),
              GET_JSON_OBJECT(extra, '$.payTime'),
              ts
          ), 1, 10) <= (SELECT end_date FROM params LIMIT 1)
      AND CAST(COALESCE(
              GET_JSON_OBJECT(extra, '$.money'),
              GET_JSON_OBJECT(extra, '$.rechargeMoney'),
              GET_JSON_OBJECT(extra, '$.price')
          ) AS DOUBLE) > 0
),
log_dedup AS (
    SELECT userid, recharge_time_str, amount_usd
    FROM (
        SELECT
            userid,
            recharge_time_str,
            amount_usd,
            ROW_NUMBER() OVER (
                PARTITION BY COALESCE(order_id, CONCAT(CAST(userid AS STRING), recharge_time_str))
                ORDER BY recharge_time_str ASC
            ) AS rn
        FROM log_raw
    ) t
    WHERE rn = 1
),
recharge_enriched AS (
    SELECT
        d.userid,
        d.amount_usd,
        HOUR(CAST(d.recharge_time_str AS DATETIME)) AS hour_of_day,
        COALESCE(NULLIF(TRIM(uf.country_name), ''), '未知') AS country_name
    FROM log_dedup d
    LEFT JOIN chamet_data.dim_user_f_d uf
        ON  d.userid = uf.userid
        AND uf.ds = (SELECT user_ds_snapshot FROM params LIMIT 1)
    WHERE HOUR(CAST(d.recharge_time_str AS DATETIME)) BETWEEN 0 AND 5
       OR HOUR(CAST(d.recharge_time_str AS DATETIME)) BETWEEN 8 AND 10
),
period_data AS (
    SELECT
        country_name,
        CASE
            WHEN hour_of_day BETWEEN 0 AND 5 THEN 'peak_0_5'
            WHEN hour_of_day BETWEEN 8 AND 10 THEN 'trough_8_10'
        END AS period_code,
        userid,
        amount_usd
    FROM recharge_enriched
),
country_period_agg AS (
    SELECT
        country_name,
        period_code,
        SUM(amount_usd) AS total_amount_usd,
        COUNT(DISTINCT userid) AS paying_user_cnt
    FROM period_data
    GROUP BY country_name, period_code
),
period_total AS (
    SELECT
        period_code,
        COUNT(DISTINCT userid) AS total_paying_users
    FROM period_data
    GROUP BY period_code
),
metrics AS (
    SELECT
        a.country_name,
        a.period_code,
        a.total_amount_usd,
        a.paying_user_cnt,
        t.total_paying_users,
        ROUND(a.total_amount_usd / a.paying_user_cnt, 2) AS arppu_usd,
        ROUND(a.paying_user_cnt * 100.0 / t.total_paying_users, 2) AS user_pct
    FROM country_period_agg a
    INNER JOIN period_total t
        ON a.period_code = t.period_code
)
SELECT
    country_name AS 国家,
    MAX(CASE WHEN period_code = 'peak_0_5' THEN paying_user_cnt END) AS 高峰期_付费用户数,
    MAX(CASE WHEN period_code = 'peak_0_5' THEN arppu_usd END) AS 高峰期_ARPPU_usd,
    MAX(CASE WHEN period_code = 'peak_0_5' THEN user_pct END) AS 高峰期_用户占比_pct,
    MAX(CASE WHEN period_code = 'trough_8_10' THEN paying_user_cnt END) AS 低谷期_付费用户数,
    MAX(CASE WHEN period_code = 'trough_8_10' THEN arppu_usd END) AS 低谷期_ARPPU_usd,
    MAX(CASE WHEN period_code = 'trough_8_10' THEN user_pct END) AS 低谷期_用户占比_pct
FROM metrics
GROUP BY country_name
ORDER BY NVL(MAX(CASE WHEN period_code = 'peak_0_5' THEN paying_user_cnt END), 0) DESC,
         country_name;
```

## 长表输出（按时段 × 国家，推荐导出）

```sql
-- 复用主查询 CTE 至 metrics，然后：
SELECT
    country_name AS 国家,
    CASE period_code
        WHEN 'peak_0_5' THEN '高峰期(0-5点)'
        WHEN 'trough_8_10' THEN '低谷期(8-10点)'
    END AS 时段,
    paying_user_cnt AS 付费用户数,
    total_paying_users AS 时段全量付费用户数,
    ROUND(total_amount_usd, 2) AS 充值总金额_usd,
    arppu_usd AS ARPPU_usd,
    user_pct AS 用户占比_pct
FROM metrics
ORDER BY period_code, paying_user_cnt DESC, country_name;
```

## 时段合计（校验占比之和 ≈ 100%）

```sql
-- 复用主查询 CTE 至 period_total
SELECT
    CASE period_code
        WHEN 'peak_0_5' THEN '高峰期(0-5点)'
        WHEN 'trough_8_10' THEN '低谷期(8-10点)'
    END AS 时段,
    total_paying_users AS 全量付费用户数,
    (SELECT COUNT(DISTINCT country_name) FROM period_data p WHERE p.period_code = t.period_code) AS 国家数
FROM period_total t;
```

## 执行注意

1. **整条 SQL 一次执行**（含全部 `WITH`）。  
2. `HOUR()` 报错时改用：  
   `CAST(SUBSTRING(recharge_time_str, 12, 2) AS BIGINT)`  
3. **用户占比**分母 = 该时段（高峰或低谷）内**所有国家**去重付费用户合计；各国占比相加应为 100%（「未知」也算一国）。  
4. 同一用户若在两时段都充值，会分别计入两个时段（占比各自独立计算）。  
5. 调整时段：改 `BETWEEN 0 AND 5` / `BETWEEN 8 AND 10` 即可（如示例图用 0～6、8～12）。  
6. `dim_user_f_d` 为日分区表；若 `20260531` 未产出，改用已产出的最新 `ds` 填入 `user_ds_snapshot`。
